四、npm优化及其更多CI环境上问题 #

4.1、npm ci #

  • 要求项目必须有package-lock.json 或者是 npm-shrinkwrap.json ，此时执行npm ci安装速度更快
  • 全量安装，不能单独安装包
  • 项目中使用了package-lock.json会提升安装时间，因为缓存里面包含了包的具体的版本信息

4.2、为什么要lock files #

  • 目的是在任意的机器上执行npm install 得到完全相同的node_modules结构
  • 如果你开发的是项目，最好提交lock 文件到仓库中，确保运维人员ci/cd正常
  • 如果是开发一个库给外部使用的话，可以复用主项目加载的包，减少依赖重复的体积

4.3、依赖管理 #

  • peerDependencies 一般是开发插件，比如开发koa插件，宿主环境肯定是有koa的才能使用, 脱离了就没有意义了
  • bundledDependencies 必须要现在dependencies和devDependencies里面声明
