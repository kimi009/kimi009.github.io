3.3、修改你的插件 #

// The module 'vscode' contains the VS Code extensibility API
// Import the module and reference it with the alias vscode in your code below
import * as vscode from 'vscode';

// This method is called when your extension is activated
// Your extension is activated the very first time the command is executed
export function activate(context: vscode.ExtensionContext) {

	// Use the console to output diagnostic information (console.log) and errors (console.error)
	// This line of code will only be executed once when your extension is activated
	console.log('Congratulations, your extension "reviewcode" is now active!');

	// The command has been defined in the package.json file
	// Now provide the implementation of the command with registerCommand
	// The commandId parameter must match the command field in package.json
	let disposable = vscode.commands.registerCommand('reviewcode.helloWorld', () => {
		// The code you place here will be executed every time your command is executed
		// Display a message box to the user
		vscode.window.showInformationMessage('恭喜你！现在可以开始代码检视啦!');
	});

	context.subscriptions.push(disposable);
}

// This method is called when your extension is deactivated
export function deactivate() {}

3.4、安装发布包 #

npm install -g @vscode/vsce

在插件根目录修改package.json添加publisher

  "name": "reviewcode",
  "publisher": "kimi009",  //这里就是上面注册的publisher
  "displayName": "代码检视",
  "description": "代码检视",
  "version": "0.0.1",
  "engines": {
    "vscode": "^1.54.0"
  },

3.5、打包 #

vsce package

  • 此时会在根目录下面生产vsix文件, 如果是内部私密插件此时可以直接给同事导入使用

3.6、发布 #

 ERROR  It seems the README.md still contains template text. Make sure to edit the README.md file before you package or publish your extension.

  • 出现上述错误的时候直接删除README.md文件内容，然后修改你插件的说明

  • 开始发布

    vsce publish
需要你的秘钥，就是上面申请的accesstoken
